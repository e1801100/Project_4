225 /*
226 *********************************************************************************************************
227 * CRITICAL SECTION CONFIGURATION
228 *
229 * Note(s) : (1) Configure CPU_CFG_CRITICAL_METHOD with CPU's/compiler's critical section method :
230 *
231 * Enter/Exit critical sections by ...
232 *
233 * CPU_CRITICAL_METHOD_INT_DIS_EN Disable/Enable interrupts
234 * CPU_CRITICAL_METHOD_STATUS_STK Push/Pop interrupt status onto stack
235 * CPU_CRITICAL_METHOD_STATUS_LOCAL Save/Restore interrupt status to local variable
236 *
237 * (a) CPU_CRITICAL_METHOD_INT_DIS_EN is NOT a preferred method since it does NOT support
238 * multiple levels of interrupts. However, with some CPUs/compilers, this is the only
239 * available method.
240 *
241 * (b) CPU_CRITICAL_METHOD_STATUS_STK is one preferred method since it supports multiple
242 * levels of interrupts. However, this method assumes that the compiler provides C-level
243 * &/or assembly-level functionality for the following :
244 *
245 * ENTER CRITICAL SECTION :
246 * (1) Push/save interrupt status onto a local stack
247 * (2) Disable interrupts
248 *
249 * EXIT CRITICAL SECTION :
250 * (3) Pop/restore interrupt status from a local stack
251 *
252 * (c) CPU_CRITICAL_METHOD_STATUS_LOCAL is one preferred method since it supports multiple
253 * levels of interrupts. However, this method assumes that the compiler provides C-level
254 * &/or assembly-level functionality for the following :
255 *
256 * ENTER CRITICAL SECTION :
257 * (1) Save interrupt status into a local variable
258 * (2) Disable interrupts
259 *
260 * EXIT CRITICAL SECTION :
261 * (3) Restore interrupt status from a local variable
262 *
263 * (2) Critical section macro's most likely require inline assembly. If the compiler does NOT
264 * allow inline assembly in C source files, critical section macro's MUST call an assembly
265 * subroutine defined in a 'cpu_a.asm' file located in the following software directory :
266 *
267 * <CPU-Compiler Directory><cpu><compiler>\
268 *
269 * where
270 * <CPU-Compiler Directory> directory path for common CPU-compiler software
271 * <cpu> directory name for specific CPU
272 * <compiler> directory name for specific compiler
273 *
274 * (3) (a) To save/restore interrupt status, a local variable 'cpu_sr' of type 'CPU_SR' MAY need
275 * to be declared (e.g. if 'CPU_CRITICAL_METHOD_STATUS_LOCAL' method is configured).
276 *
277 * (1) 'cpu_sr' local variable should be declared via the CPU_SR_ALLOC() macro which, if
278 * used, MUST be declared following ALL other local variables.
279 *
280 * Example :
281 *
282 * void Fnct (void)
283 * {
284 * CPU_INT08U val_08;
285 * CPU_INT16U val_16;
286 * CPU_INT32U val_32;
287 * CPU_SR_ALLOC(); MUST be declared after ALL other local variables
288 * :
289 * :
290 * }
291 *
292 * (b) Configure 'CPU_SR' data type with the appropriate-sized CPU data type large enough to
293 * completely store the CPU's/compiler's status word.
294 *********************************************************************************************************
